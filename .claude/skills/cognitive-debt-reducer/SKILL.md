# 認知負荷の低減

コードや概念の構造を理解し、認知負荷を低減するためのスキル。リニアなウォークスルードキュメントの生成と、アルゴリズムをアニメーションで可視化するインタラクティブHTMLの生成によってこれを達成する。

---

## 背景・目的

AIエージェントが生成したコードは動くが、**なぜそう動くのかわからなくなる（Cognitive Debt）**。
テキストの解説だけでは直感的な理解が得られないケースが多い。

特にアルゴリズムや数値的な処理（ソート・グラフ探索・座標計算・最適化など）は、
アニメーションで可視化することで「一目で分かる」理解が得られる。

このスキルは以下の流れで認知的負債を返済する：

1. **リニアなウォークスルー生成** — コードの構造をテキストで説明するドキュメントを作る
2. **アニメーション付きHTML生成** — ウォークスルーを元にアルゴリズムを可視化するHTMLを作る

---

## トリガー

以下のような発言でこのスキルを起動する：

- 「このコードを可視化して」
- 「インタラクティブな説明ページを作って」
- 「このアルゴリズムを可視化して」
- 「どう動くか見えるようにして」
- 「interactive explainer を作って」

```
.claude/skills/cognitive-debt-reducer/
├── SKILL.md
└── output/
    └── YYYYMMDD-<slug>/
        ├── walkthrough.md          ← Step 1: リニアなコード解説

```

- `YYYYMMDD`: 実行日の日付
- `<slug>`: 対象コードやアルゴリズム名をケバブケースに変換（例: `word-cloud-placement`）

## 実行手順

### 0. 対象の特定

以下の優先順で対象コードを取得する：

**A. 会話コンテキストから（最優先）**
- 現在の会話にコードやファイルパスが含まれていればそれを使う

**B. ファイルパスの指定**
- ユーザーがパスを指定した場合は Read ツールで読み込む

**C. URLの指定**
- URLが指定された場合は以下で取得する：
  ```bash
  npx -y @mizchi/readability --format=md <url>
  ```

**D. ユーザーに確認**
- 不明な場合はコードの貼り付けまたはパスの指定をユーザーに依頼する

---

### 1. HTML生成の確認

対象コードの取得が完了したら、**必ずユーザーに以下の確認を取ること**：

```
対象コードを確認しました。
インタラクティブHTMLの生成も行いますか？（ウォークスルーのみより多くのトークンがかかります）

[y] はい、ウォークスルー + HTML を生成する
[n] いいえ、ウォークスルーだけ生成する

y または n で答えてください。
```

- **n と答えた場合**: Step 2（ウォークスルー生成）のみ実施し、Step 3（完了報告）へ進む。HTMLパスは報告から省略する。
- **y と答えた場合**: Step 2 → Step 3 → Step 4 の順に進む。

---

### 2. リニアなウォークスルーの生成

対象コードを読み込み、以下の構成でウォークスルードキュメントを作成する。

**ウォークスルーの構成:**

```markdown
# Walkthrough: <タイトル>

## 概要

<このコード/アルゴリズムが何をするか。2〜3文で>

## 処理の流れ（線形ウォークスルー）

### Step 1: <最初の処理名>
<何をしているか。なぜそうするか>

### Step 2: <次の処理名>
...

## 核心となるデータ構造

<アルゴリズムの中心にあるデータ構造や状態を説明>

## 可視化のポイント

<アニメーションで見せると効果的な部分のリスト>
- ...
```

Write ツールで `.claude/skills/cognitive-debt-reducer/output/YYYYMMDD-<slug>/walkthrough.md` に保存する。

---

### 3. インタラクティブHTML説明ページの生成

> Step 1 で **n** と答えた場合はこのステップをスキップして Step 4 へ進む。

ウォークスルーをもとに、アルゴリズムの動作をアニメーションで可視化するHTMLファイルを生成する。

**必須機能（全ページ共通）:**

| 機能 | 説明 |
|------|------|
| アニメーション表示 | アルゴリズムの各ステップをキャンバスまたはSVGで可視化 |
| 再生/一時停止ボタン | スペースキーでもトグル可能にする |
| ステップスライダー | 任意のフレームにジャンプできるレンジ入力 |
| コマ送りボタン | 一時停止中に1ステップずつ進める・戻るボタン |
| 速度調整スライダー | アニメーション速度を0.25x〜4xの範囲で調整 |
| ステップ説明テキスト | 現在のステップで何が起きているかを日本語で表示 |

**推奨機能（コンテキストに応じて判断）:**

| 機能 | 説明 |
|------|------|
| テキスト入力 | ユーザーが任意のデータを入力して試せる場合は実装 |
| URLフラグメント永続化 | 入力データを `#` フラグメントに保存してリロード後も復元 |
| PNG書き出し | 現在の可視化状態をPNG画像としてダウンロード |
| ダーク/ライトテーマ | 閲覧環境に合わせて切り替え |

**実装ガイドライン:**

- **単一HTMLファイル** として完結させる（外部CDN依存なし、またはCDNを明示）
- JavaScript: バニラJS（TypeScriptコンパイル不要）で書く
- スタイル: インラインCSS、またはTailwind CDNを使用
- Canvas APIを使う場合は `requestAnimationFrame` でスムーズに描画する
- 各アニメーションフレームの状態を配列として事前計算しておき、スライダーで任意フレームにジャンプできるようにする
- アニメーション速度は `setInterval` または `setTimeout` のdelayを変更して実装
- モバイル対応（最低限タッチイベントの考慮）

**HTMLの基本構造:**

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title><アルゴリズム名> - インタラクティブ説明</title>
  <!-- スタイルをここに記述 -->
</head>
<body>
  <!-- ① ヘッダー: タイトルと概要説明 -->
  <!-- ② 入力エリア（オプション）: テキスト入力やパラメータ設定 -->
  <!-- ③ 可視化エリア: Canvas または SVG -->
  <!-- ④ ステップ説明テキスト: 現在ステップの解説 -->
  <!-- ⑤ コントロールパネル: 再生・停止・スライダー・速度 -->
  <!-- ⑥ スクリプト -->
</body>
</html>
```

Write ツールで `.claude/skills/cognitive-debt-reducer/output/YYYYMMDD-<slug>/<slug>-explainer.html` に保存する。

**HTMLを保存したら、デフォルトブラウザで自動オープンする:**

```bash
# macOS / Linux 共通: open コマンドがデフォルトブラウザで開く
open ".claude/skills/cognitive-debt-reducer/output/YYYYMMDD-<slug>/<slug>-explainer.html"
```

> `open` コマンドは macOS ではデフォルトブラウザ、Linux では `xdg-open` に相当する。
> パスは実際に保存したファイルの絶対パスまたは相対パスを使用すること。

---

### 4. 完了報告

```
## インタラクティブ説明ページ生成完了

**対象**: <アルゴリズム/コード名>

**生成ファイル:**
- ウォークスルー: `.claude/skills/cognitive-debt-reducer/output/YYYYMMDD-<slug>/walkthrough.md`
- HTMLページ: `.claude/skills/cognitive-debt-reducer/output/YYYYMMDD-<slug>/<slug>-explainer.html`

**可視化のポイント:**
- <アニメーションで見えるようになったポイント1>
- <アニメーションで見えるようになったポイント2>

**使い方:**
ブラウザで HTML ファイルを開いてください。再生ボタンで自動再生、スライダーで任意のステップにジャンプできます。
（HTMLを生成した場合はブラウザが自動で開きます）
```

HTMLを生成しなかった場合は、HTMLパスの行を完了報告から省略する。

---

## アルゴリズム別の可視化ヒント

| アルゴリズム種別 | 可視化方法 |
|----------------|-----------|
| ソートアルゴリズム | 棒グラフの高さで値を表し、比較・交換をハイライト |
| グラフ探索（BFS/DFS） | ノードとエッジをSVGで描画し、訪問済みノードを色変え |
| 座標配置（ワードクラウド等） | 2Dキャンバスに配置試行をリアルタイム描画 |
| 動的計画法 | DPテーブルをグリッドで描画し、セル埋めをアニメーション |
| テキスト処理（トークナイザ等） | 入力文字列をスパンで分割してハイライト |
| ベクトル/埋め込み | 2D/3D散布図でクラスタリングや近傍を可視化 |
| LLMデコーディング（Beam Search等） | ツリー構造でビームをアニメーション展開 |

---

## 注意事項

- ウォークスルー生成後、必ず内容を確認してからHTML生成に進む（二段階確認）
- HTMLファイルは **ブラウザでそのまま開けること** を必ず確認する（相対パス参照など開けない構造は禁止）
- 外部リソース（CDN）を使う場合はオフライン不可である旨をコメントに明記する
- アニメーションのフレーム数が多い場合（1000フレーム超）は事前計算をWebWorkerで行うか、遅延計算にする
- ディレクトリ作成は Bash で `mkdir -p` を実行する
- ファイル作成は必ず Write ツールを使う
